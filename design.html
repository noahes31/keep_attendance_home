Final Engineering Design

The architecture, components, and system flow behind Keep Attendance.

System Overview

Keep Attendance is built on three tightly integrated components — camera hardware, an AI analysis pipeline, and a web interface.

The system uses a standard network camera as its input device, requiring no specialized hardware. The camera streams live video to a local processing server running a machine learning model trained to detect and verify human presence.

When presence is confirmed above a confidence threshold, the system creates a timestamped attendance record and makes it immediately available through a lightweight web dashboard. All processing happens locally — no cloud dependency, no latency, and no data leaving the premises.

The web interface is accessible from any browser on the same local network, making it easy for administrators to monitor attendance in real time without installing any software.

InputNetwork camera (IP/USB)
BackendPython · ML model · local server
OutputTimestamped attendance log
InterfaceBrowser-based web dashboard
NetworkLocal network — no cloud required
Hardware costLow — standard consumer devices
