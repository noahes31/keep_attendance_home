Ideate Solution

From problem to concept — how we developed the idea for Keep Attendance.

The Ideation Process

With a clear problem in hand, we began exploring how technology could fundamentally change the way attendance is tracked. Rather than digitizing the same manual process, we asked: what would it look like if attendance happened passively — without any human action required?

This question led us toward computer vision and machine learning. If a camera could reliably detect whether a person is present in a space, attendance could become a background process — always running, always accurate, always available.

We evaluated several approaches before converging on an architecture that was low-cost, portable, and accessible — qualities that would make the system viable beyond the prototype stage.

Approaches We Considered

We evaluated multiple solutions before choosing the AI vision approach.

Considered

QR Code Check-In

Students scan a QR code to log attendance. Simple to implement but still requires manual action and is easy to game.

Considered

RFID / NFC Tags

Hardware tags detect presence automatically. Higher cost, requires physical infrastructure, and isn't easily portable.

Considered

Mobile App Check-In

App-based self-reporting. Still manual, relies on devices being present, and doesn't provide passive verification.

Chosen

AI Computer Vision

A network camera + machine learning model detects presence passively. No user action required. Portable, low-cost, and fully automated.

Design Thinking Framework

We followed a structured design thinking process from empathy to prototype.

1

Empathize

Identified the real pain points of manual attendance from an educator and administrator perspective.

2

Define

Narrowed the problem statement: automated, passive, reliable attendance for any space with a camera.

3

Ideate

Generated multiple solution concepts and evaluated each against cost, reliability, and accessibility criteria.

4

Prototype

Built an initial working system with a network camera, a Python ML backend, and a simple web dashboard.

5

Test

Ran the system in real environments, measured detection accuracy, and iterated on the model and UI.
